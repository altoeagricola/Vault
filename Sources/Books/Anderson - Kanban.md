Internal sources of variation are variations that are under the control of the system in operation. With Kanban for software engineering and IT operations, we think of the system as a process that is defined by a set of policies that govern the system operation. They can be directly affected by changes made by individuals, the team, or management. Changes to policies change the system operation and its performance. Therefore, a change to the process definition represents a change that affects the internal sources of variation. — location: [3925](kindle://book?action=open&asin=B0057H2M70&location=3925) ^ref-60857
