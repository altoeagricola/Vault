PARA is an organizational framework created by [[Tiago Forte]] that sorts all information into four categories based on *actionability* — how close the material is to producing an outcome. The acronym stands for Projects, Areas, Resources, and Archives.

## The four categories

| Category | Definition | Actionability |
|----------|-----------|---------------|
| **Projects** | Short-term efforts with a defined goal and deadline | Highest — active now |
| **Areas** | Ongoing responsibilities with a standard to maintain | High — no end date, but require attention |
| **Resources** | Topics of interest or reference material | Medium — useful but not urgent |
| **Archives** | Inactive items from the other three categories | Low — preserved, not active |

## The key insight

PARA organizes by *how you'll use* information, not by *what it's about*. A note about Docker might live in Projects (if you're building something with it this week), Areas (if you're responsible for container infrastructure), or Resources (if you're just learning about it). The same content moves between categories as its actionability changes.

## Why it works

Most organizational systems break down because they try to create a "correct" taxonomy upfront. PARA sidesteps this by accepting that the *purpose* of information changes over time. The four buckets are simple enough to maintain without deliberation — when you touch a piece of information, you know immediately which of the four categories it belongs to right now.

## PARA and GTD

PARA complements [[Getting Things Done]] rather than replacing it. GTD provides the *workflow* — capture, clarify, organize, reflect, engage — while PARA provides the *structure* for where organized items land. Many [[Personal Knowledge Management]] systems blend both: GTD's process discipline with PARA's organizational clarity.

## Adaptations

Practitioners often modify PARA to fit their context. Common variations include renaming "Resources" to something more specific, adding subcategories within Areas, or using the framework across multiple tools (notes, files, email). The structure is deliberately minimal so it can be adapted without breaking.
